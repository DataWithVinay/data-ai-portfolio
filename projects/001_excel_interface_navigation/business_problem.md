# Business Problem — Retail Operations Workbook Navigation

## Context
A retail company receives daily sales and stock audit data from multiple stores. The data is shared as spreadsheets, but every store uses slightly different sheet names and file structures.

## Problem
The analytics team loses time every week because:
- sheet names are unclear
- headers are inconsistent
- store/product lookup data is mixed with transactions
- filtering and navigation are slow
- there is no README/data dictionary

## Business Objective
Design a beginner-friendly, clean Excel workbook structure that can be used for:
- sales review
- product lookup
- inventory audit
- future PivotTables
- future Power Query transformations
- future Power BI dashboarding

## Stakeholders
- Store operations manager
- Inventory analyst
- Sales analyst
- Regional manager
- Data analyst

## KPIs
- Time to locate required sheet
- Number of manually fixed column names
- Number of audit mismatches found
- % completed orders
- Total net sales by store/region

## Deliverables
- Synthetic sample datasets
- Excel navigation learning notes
- GitHub-ready project folder
- LinkedIn educational post
- Image prompts
- PDF-ready revision notes
