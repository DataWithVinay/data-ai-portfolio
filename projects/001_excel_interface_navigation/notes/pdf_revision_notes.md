# PDF-Ready Revision Notes — Day 01

## Topic
Excel Interface, Navigation & Workbook Structure

## Key Concepts
- Workbook = full Excel file
- Worksheet = tab inside workbook
- Cell = single box
- Range = group of cells
- Formula bar = view/edit selected cell content
- Name box = jump to cell/range
- Freeze panes = keep headers visible
- Filters = narrow visible data
- Clean sheet names = business clarity

## Best Practices
1. Use one dataset per worksheet.
2. Use clear sheet names.
3. Freeze header rows.
4. Apply filters.
5. Avoid merged cells in raw data.
6. Keep raw data, calculations, dashboards, and notes separate.
7. Maintain a README or data dictionary sheet.

## Common Mistakes
- Vague sheet names
- Blank rows inside data
- Merged cells in source tables
- Mixed raw and calculated data
- No data dictionary

## Interview Revision
Q: Why are merged cells bad in raw data?
A: They break filters, sorting, formulas, Power Query imports, and automation.

Q: What makes an Excel file analytics-ready?
A: Clean headers, consistent columns, one dataset per sheet, no blank rows, and clear sheet names.
